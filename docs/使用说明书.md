# 天枢系统 HUD — 使用说明书

> 版本：v3.1 · 适用项目：[tianshu-hub](https://github.com/lin-001-007/tianshu-hub)  
> 文档目标：零基础也能按步骤完整部署、启动、使用与排错。

---

## 1. 系统概述

**天枢 HUD** 是一套独立的 Windows 桌面系统遥测大屏，以科幻 HUD 风格实时展示 CPU、内存、磁盘、网络、GPU、温度、进程等 **16+ 项指标**，并附带：

- 反应堆全息动画（负载/温度联动）
- 多维度图表（柱图、波形、雷达、环形图、趋势 sparkline）
- Windows 系统事件日志 + 遥测动态日志
- 可选 AI 将英文/报错类日志概括成中文

**重要说明：**

| 项目 | 说明 |
|------|------|
| 运行形态 | 本地浏览器窗口或全屏 Kiosk，**不会修改 Windows 桌面壁纸文件** |
| 独立性 | 纯本地项目，不依赖 Docker、数据库或其他 Agent 项目 |
| 数据范围 | 仅采集本机性能与系统日志，不上传云端（AI 日志翻译除外，需自行配置密钥） |

## 3. 环境要求

### 3.1 操作系统

- **Windows 10 / 11**（64 位，推荐）
- 日志中的 Windows 事件采集依赖 PowerShell `Get-WinEvent`

### 3.2 必备软件

| 软件 | 版本建议 | 用途 | 验证命令 |
|------|----------|------|----------|
| Python | 3.9+ | 后端 API | `python --version` |
| Node.js | 18+ | 前端 dev/build | `node --version` |
| npm | 随 Node 安装 | 前端依赖 | `npm --version` |
| Microsoft Edge | 任意较新版本 | 一键打开 HUD | 系统自带 |

### 3.3 可选组件

| 组件 | 用途 |
|------|------|
| DashScope API Key | 英文 Windows 日志 AI 中文概括 |
| NVIDIA 驱动 + pynvml | 更准确的 NVIDIA GPU 负载（未安装时走 Windows 性能计数器） |
| Lively Wallpaper | 将 HUD 网页设为动态壁纸 |

### 3.4 端口占用

启动前请确认以下端口未被其他程序长期占用：

| 端口 | 服务 |
|------|------|
| **8799** | 指标 API + SSE 流 |
| **5199** | HUD 前端（Vite 开发服务器） |

检查命令：

```powershell
netstat -ano | findstr ":8799 :5199"
```

## 4. 首次安装（逐步可复现）

以下命令在 **PowerShell** 中执行。

### 4.1 进入项目目录

```powershell
cd tianshu-hub
```

### 4.2 安装 Python 依赖

```powershell
python -m pip install -r .\server\requirements.txt
```

依赖清单（`server/requirements.txt`）：

- fastapi
- uvicorn
- psutil
- httpx
- python-dotenv

验证安装：

```powershell
python -c "import fastapi, psutil, uvicorn, httpx; print('Python deps OK')"
```

### 4.3 安装前端依赖

```powershell
cd .\web
$env:NO_PROXY = "*"
npm install --registry https://registry.npmmirror.com
cd ..
```

验证：

```powershell
Test-Path .\web\node_modules
# 应输出 True
```

### 4.4 配置 AI 日志翻译（可选）

若希望英文/报错类 Windows 事件自动概括成中文：

```powershell
Copy-Item .\server\.env.example .\server\.env
notepad .\server\.env
```

编辑 `.env`，填入你的 DashScope 密钥：

```env
DASHSCOPE_API_KEY=你的密钥
HUD_LOG_MODEL=qwen-turbo
```

> 不配置 `.env` 也可正常使用 HUD；日志仍会显示，但英文事件主要依赖内置规则翻译，复杂报错可能保留原文。

## 5. 启动方式

### 5.1 方式 A：一键启动（推荐）

**普通可调整窗口**（默认）：

```powershell
cd tianshu-hub
.\start-hud.ps1
```

**全屏 Kiosk 模式**（适合当桌面背景）：

```powershell
.\start-hud.ps1 -Kiosk
```

脚本会自动完成：

1. 读取 `server\.env` 中的 `DASHSCOPE_API_KEY`
2. 检测并安装缺失的 Python / npm 依赖
3. 在 **8799** 启动指标 API（若旧进程无 GPU 字段会自动重启）
4. 在 **5199** 启动 Vite 前端（若未运行）
5. 用 Edge 打开 `http://127.0.0.1:5199`（Kiosk 模式带 `?desktop=1`）

成功时终端输出：

```
OK: HUD running. API :8799 | Web :5199
```

### 5.2 方式 B：手动分步启动（便于调试）

**终端 1 — 启动后端：**

```powershell
cd tianshu-hub\server
python -m uvicorn main:app --host 127.0.0.1 --port 8799
```

**终端 2 — 启动前端：**

```powershell
cd tianshu-hub\web
$env:NO_PROXY = "*"
npm run dev
```

**浏览器访问：**

```
http://127.0.0.1:5199
```

全屏桌面模式 URL：

```
http://127.0.0.1:5199/?desktop=1
```

### 5.3 方式 C：生产构建 + 静态预览

```powershell
cd tianshu-hub\web
npm run build
npm run preview
```

> 生产模式下前端会直接请求 `http://127.0.0.1:8799`，后端必须单独运行。

## 6. 界面布局说明

HUD 采用 **顶栏 + 三栏主区 + 底栏** 结构，各分区可通过拖拽分隔条调整大小，布局自动保存到浏览器 `localStorage`。

```
┌─────────────────────────────────────────────────────────────┐
│ 顶栏：品牌 | CPU 环 gauge | 链路状态 | 时钟                    │
├──────────┬──────────┬───────────────────────────────────────┤
│ 左栏     │ 中栏     │ 右栏（自适应剩余宽度）                  │
│ 反应堆   │ 关键传感 │ 数据分析引擎                           │
│ 内存资源 │ CPU 柱图 │ 遥测指标矩阵（16+ 卡片）                │
│          │ 系统日志 │                                       │
├──────────┴──────────┴───────────────────────────────────────┤
│ 底栏左：系统负载监测（柱图+波形） | 底栏右：系统概览（雷达等）   │
└─────────────────────────────────────────────────────────────┘
```

### 6.1 各区域功能

| 区域 | 内容 |
|------|------|
| **顶栏 CPU 环** | 实时 CPU 总负载百分比，环带长度随负载变化 |
| **反应堆全息** | 线框核心动画；亮度与 CPU 温度/GPU 负载联动 |
| **反应堆下方数值** | GPU 负载、显卡名称、CPU 负载、综合负载指数 |
| **内存资源 MEMORY** | 环形图 + 已用/可用/交换区等进度条 |
| **关键传感 QUICK READ** | CPU、内存、磁盘、网络等核心指标横向条 |
| **CPU 核心负载** | 每个逻辑核心独立柱图（C0、C1…） |
| **系统重要日志** | Windows 事件 + 遥测动态；可展开英文原文 |
| **数据分析引擎** | 扫描进度条 + 迷你统计 |
| **遥测指标矩阵** | 全部指标卡片，3 列网格 + 进度条 |
| **系统负载监测** | 底部柱图与波形动画 |
| **系统概览** | 雷达图、横向 gauge、详情卡片、网络流量趋势 |

### 6.2 拖拽调整布局

| 分隔条位置 | 调整内容 | localStorage 键 |
|------------|----------|-----------------|
| 左栏与中栏之间 | 反应堆栏宽度 | `hud-reactor-w` |
| 中栏与右栏之间 | 中间栏宽度 | `hud-mid-w` |
| 主区与底栏之间 | 底栏高度 | `hud-bottom-h` |
| 底栏左右之间 | 底栏左半宽度 | `hud-bottom-left-w` |

**恢复默认布局：** 浏览器控制台执行：

```javascript
localStorage.removeItem('hud-reactor-w')
localStorage.removeItem('hud-mid-w')
localStorage.removeItem('hud-right-w')
localStorage.removeItem('hud-bottom-h')
localStorage.removeItem('hud-bottom-left-w')
location.reload()
```

### 6.3 全屏桌面模式

- 启动：`.\start-hud.ps1 -Kiosk` 或访问 `?desktop=1`
- 按 **Esc** 退出全屏
- 屏幕底部会提示「按 Esc 退出全屏桌面模式」

### 6.4 链路状态指示

顶栏右侧显示 SSE 连接状态：

| 状态 | 含义 |
|------|------|
| 连接中 | 正在建立 `/api/stream` |
| 已连接 | 指标每秒推送正常 |
| 重连中 | 后端断开，自动退避重连 |

## 7. 指标说明

数据由 `psutil` + Windows 性能计数器/WMI 采集，经 `/api/metrics` 或 SSE 推送，刷新频率约 **1 秒**。

### 7.1 核心指标列表

| 指标 ID | 显示名称 | 单位 | 说明 |
|---------|----------|------|------|
| cpu | CPU 总负载 | % | 全核平均使用率 |
| cpu_freq | CPU 频率 | MHz | 当前频率 |
| cpu_load | 系统负载 | — | 1 分钟 load average |
| cpu_temp | CPU 核心温度 | °C | 优先读传感器；无传感器时按负载估算（38–80°C） |
| gpu | GPU 负载 | % | NVIDIA NVML 或 Windows GPU Engine 计数器 |
| gpu_mem | GPU 显存 | % | 有 NVIDIA 或计数器支持时显示 |
| mem | 内存占用 | % | 物理内存使用率 |
| mem_used | 已用内存 | GB | |
| mem_avail | 可用内存 | GB | |
| swap | 交换分区 | % | 页面文件/交换区 |
| disk | 磁盘使用 | % | 系统盘占用 |
| disk_r / disk_w | 磁盘读/写 | MB/s | 与上次采样间隔计算速率 |
| disk_free | 磁盘剩余 | GB | |
| net_up / net_down | 网络上传/下载 | MB/s | |
| net_conn | 网络连接 | — | TCP/UDP inet 连接数 |
| proc / thread | 进程数/线程数 | — | |
| uptime | 运行时长 | h | 自上次开机 |
| battery | 电池 | % | 笔记本有电池时显示 |
| temp_* | 温度传感器 | °C | 硬件支持时追加 |

### 7.2 反应堆区域术语

| 显示项 | 含义 |
|--------|------|
| **GPU 负载** | 显卡 3D/计算引擎利用率 |
| **综合负载指数** | 综合 CPU、内存、磁盘、网络等加权后的整体繁忙程度（非辐射/核辐射） |
| **核心亮度** | 随合理温度范围（约 35–100°C）或 GPU 负载变化 |

### 7.3 GPU 显示「未检测到」时

按顺序排查：

1. 运行 `.\start-hud.ps1` 让脚本检测 API 是否含 `gpu` 字段，必要时自动重启 8799
2. 确认本机有独立/核显且驱动正常
3. 手动验证 API：

```powershell
(Invoke-RestMethod http://127.0.0.1:8799/api/metrics).data.gpu | Format-List
```

4. 若仍异常，清除布局缓存后刷新（见 6.2）

## 8. 日志系统

### 8.1 日志来源

1. **遥测动态**：根据 CPU/内存/磁盘/网络突变自动生成中文提示  
2. **Windows 事件**：从 System、Application 日志读取 Level 1–3（严重/错误/警告）事件

### 8.2 刷新机制

| 通道 | 路径 | AI 翻译 | 频率 |
|------|------|---------|------|
| SSE 主流 | `/api/stream` | 否（保证实时性） | ~1 秒 |
| 日志轮询 | `/api/logs` | 是（若配置密钥） | 2 秒 |

日志卡片上显示 **「已翻译」** 标签表示经过规则或 AI 概括。

### 8.3 查看英文原文

点击日志卡片下方 **「查看英文原文」** 可展开原始 Windows 事件文本。

## 9. API 接口（可复现验证）

后端基址：`http://127.0.0.1:8799`

### 9.1 健康检查

```powershell
Invoke-RestMethod http://127.0.0.1:8799/health
```

期望：`status = ok`

### 9.2 一次性获取指标 + 卡片 + 日志

```powershell
$r = Invoke-RestMethod http://127.0.0.1:8799/api/metrics
$r.data.cpu.percent
$r.cards.Count
$r.logs.Count
```

### 9.3 仅获取日志（含 AI）

```powershell
(Invoke-RestMethod http://127.0.0.1:8799/api/logs).logs | Select-Object -First 3
```

### 9.4 SSE 实时流

```powershell
curl -N http://127.0.0.1:8799/api/stream
```

每行格式：`data: {"type":"metrics","data":{...},"cards":[...],"logs":[...]}`

## 10. 作为桌面背景使用

天枢 HUD **不会替换 Windows 壁纸文件**，以下两种方式可实现「动态桌面」效果：

### 10.1 Edge Kiosk 全屏（内置）

```powershell
.\start-hud.ps1 -Kiosk
```

- 全屏覆盖桌面之上
- Esc 退出
- 需保持 PowerShell 窗口运行（后端/前端 Job）

### 10.2 Lively Wallpaper

1. 安装 [Lively Wallpaper](https://github.com/rocksdanister/lively)
2. 先运行 `.\start-hud.ps1` 确保 5199 与 8799 在线
3. Lively 中添加 **网页壁纸**，URL 填：`http://127.0.0.1:5199/?desktop=1`
4. 开机自启需在 Lively 与 `start-hud.ps1` 中分别设置

## 11. 停止服务

### 11.1 一键启动后

在运行 `start-hud.ps1` 的 PowerShell 窗口按 **Ctrl+C**。

### 11.2 手动清理端口

```powershell
# 结束占用 8799 的进程
Get-NetTCPConnection -LocalPort 8799 -State Listen -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force -ErrorAction SilentlyContinue }

# 结束占用 5199 的进程
Get-NetTCPConnection -LocalPort 5199 -State Listen -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force -ErrorAction SilentlyContinue }
```

## 12. 故障排查

### 12.1 页面空白或编译错误

1. 查看运行 `npm run dev` 的终端是否有 Vue/CSS 报错  
2. 浏览器 F12 → Console 查看前端错误  
3. 尝试：

```powershell
cd tianshu-hub\web
Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue
npm run dev
```

### 12.2 指标不更新 / 显示「重连中」

1. 确认后端存活：

```powershell
Invoke-WebRequest http://127.0.0.1:8799/health -UseBasicParsing
```

2. 若失败，重启后端（见 5.2 方式 B）  
3. 检查防火墙是否拦截本机 127.0.0.1 回环

### 12.3 Vite 代理报错 `ECONNREFUSED 127.0.0.1:8799`

**原因：** 前端已启动，后端未启动或已崩溃。  
**处理：** 先启动 `uvicorn`，再访问前端。

### 12.4 GPU 一直「未检测到」

**原因：** 旧 API 进程未重启，或驱动/计数器不可用。  
**处理：** 运行 `.\start-hud.ps1`（会自动探测并重启 API）。

### 12.5 日志全是英文

**原因：** 未配置 `DASHSCOPE_API_KEY`，且事件不匹配内置规则。  
**处理：** 配置 `server\.env` 后重启 API。

### 12.6 `npm install` 很慢或失败

使用国内镜像：

```powershell
npm install --registry https://registry.npmmirror.com
```

### 12.7 删除项目文件夹提示「文件正在使用」

常见占用源：

- 5199/8799 对应的 node / python 进程  
- Docker Desktop 若曾挂载该目录  

先停止 HUD 服务（见第 11 节），再删除。
